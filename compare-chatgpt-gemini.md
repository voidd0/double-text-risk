# double-text-risk vs ChatGPT or Gemini

`double-text-risk` is not trying to replace a general-purpose model.

It is trying to win a narrower, higher-frequency moment:

> "I already drafted the follow-up. I just need to know whether it adds value, should wait, should switch channels, or should stay unsent."

That is where a deterministic checker beats a blank prompt box.

## When double-text-risk is the better first step

- You want a stable `send now`, `wait longer`, `switch channels`, or `do not send` call.
- You do not want to write prompts, invent context, or compare three drifting model answers.
- You are checking a dating, recruiter, client, support, family, sales, or workplace follow-up where timing and leverage matter more than brainstorming.
- You want the same draft to score the same way every time.

## When ChatGPT or Gemini is still useful

- You want rewrites, roleplay, or broader brainstorming.
- You want a general assistant rather than a narrow follow-up timing call.
- You need long-form drafting more than disciplined decision support.

## The practical difference

| question | double-text-risk | ChatGPT / Gemini |
| --- | --- | --- |
| "Should I send this follow-up now or wait?" | deterministic action + why | depends on prompt and model variance |
| "Can I re-check the same draft later?" | yes, same input = same output | not reliably |
| "Do I need an account or API key?" | no | usually yes |
| "Can it brainstorm five different rewrites?" | no, narrow checker by design | yes |
| "Can it read the deeper recurring pattern behind this person?" | no, move into tells for that | not from one prompt alone |

## What each tool is actually good at

- Use `double-text-risk` when the draft already exists and the real question is timing, pressure, value, or channel shift.
- Use ChatGPT or Gemini when the user wants brainstorming, roleplay, or rewrite variants more than a deterministic go/wait call.
- Use `ghost-or-go` when the silence itself is the question and you need a stable `wait`, `one final ping`, or `stop` call.
- Use `message-next-step` when the silence already broke and the real decision is how to handle the first inbound reply.
- Use `replytone` when the timing is fine but the user wants to know whether the draft sounds warm, clear, or pressuring.
- Use `call-not-text` when the better move may be a channel shift rather than another ping.
- Use `ambiguity-meter` when the user's follow-up anxiety is really a mixed-signals problem.
- Use `tells` when the problem is no longer one follow-up but a recurring silence pattern, one loaded thread, or one person whose behaviour needs wider context.
